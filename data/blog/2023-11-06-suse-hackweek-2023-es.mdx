## ¿Qué es la Hackweek?

Hack Week es el momento en el que los empleados de SUSE experimentan, innovan y aprenden sin interrupciones durante toda una semana. En equipo o en solitario, pero siempre sin límites.
Una tradición de SUSE desde 2007, Hack Week ha reunido a cientos de hackers de todo el mundo para colaborar en el código abierto. Más información sobre la Hack Week.

## Objetivos de la Hackweek 2023

  - Necesita tener despliegues automatizados

## Hackweek Día 1 - Crear un sitio web alojado en páginas de GitHub

### Motivación

Durante mucho tiempo mantuve un blog personal en WordPress, pero lo desconecté porque no tenía tiempo para mantenerlo. Recientemente, empecé a escribir un diario y estoy retomando mi interés por la escritura.
La Hackweek me pareció una oportunidad excelente para volver a escribir, ya que pretendo hacer un seguimiento de mis progresos en este blog.
Pero esta vez, quería algo distinto a un blog de WordPress con un tema ya hecho. Quería algo personalizable y de código abierto, y también necesitaba tener despliegues automatizados. Quería alojarlo en GitHub y aprovechar las páginas de GitHub.

### Next.js

Antes de mis recientes aventuras como Project Manager, trabajé como Diseñador/Desarrollador de UI, trabajando con muchos frameworks diferentes a lo largo de los años. Uno de ellos es React, con el que estoy familiarizado. Next.js es un framework de React con funciones adicionales, como la generación de páginas estáticas. Era la plataforma adecuada para trabajar en mi caso de uso.

### Elegir una plantilla y personalizarla

Debido a las limitaciones de tiempo, crear un tema desde cero era una mala idea. Me puse a buscar una plantilla. Hay muchas plantillas de código cerrado para Next.js, pero yo necesitaba una con licencia de código abierto. Por suerte, encontré la plantilla [Tailwind Nextjs Starter Blog](https://github.com/timlrx/tailwind-nextjs-starter-blog), que utiliza Next.js y el framework CSS Tailwind. Es una base excelente para un blog personalizable.

Pasé a crear un logotipo con Figma y personalizar la plantilla un poco. Voy a añadir más ajustes a la misma en breve, ya que es actualmente bastante "vanilla".

### Alojamiento en GitHub

Dado que Next.js genera sitios estáticos rápidamente, configurarlo para el autoalojamiento fue pan comido.Decidí no bifurcar el proyecto, sino simplemente descargarlo y añadirlo a un nuevo repositorio. En retrospectiva, debería haberlo bifurcado; mi proyecto se habría desviado significativamente de la corriente principal. El tiempo lo dirá.

Configurar las páginas de Github y apuntar un nombre de dominio a ellas es extremadamente fácil.Utilizo Cloudflare para gestionar los DNS y seguí las guías de GitHub para configurarlo todo. ¡Facilísimo!

### Despliegues automáticos

Esto fue, de nuevo, extremadamente fácil de configurar.Github [tiene una acción](https://github.com/jhmarina/zenon-blog/actions/workflows/nextjs.yml) para publicar un sitio Next.js en GitHub Pages.Todo lo que tuve que hacer fue seguir las instrucciones de la plantilla para ajustar algunas cosas, ¡y funcionó! Cada vez que fusiono un commit al repositorio remoto, se encarga del despliegue automáticamente.

### Conclusión

Estoy satisfecho con el resultado.Tengo un blog alojado en GitHub; es de código abierto y fácilmente personalizable.Lo hice todo en unas pocas horas, lo que me permitió centrarme en otros proyectos de la Hackweek.En definitiva, ¡todo un éxito!
